Day 5-6: Bug Origin Tracing
What to Build:

Line-level commit tracing
Evolution chain tracking
Commit diff extraction

AI Prompts:
PROMPT 12 (Bug Tracing Algorithm):
"Implement GitArchaeologyService.trace_bug_origin(repo_path, file_path, line_number):
1. Use git blame to find the commit that last modified the line
2. Extract commit details: hash, author (anonymized), date, message
3. Get the diff for that commit showing the actual change
4. Use 'git log --follow -L{line},{line}:{file}' to trace line history across renames
5. Build evolution chain: list of all commits that touched this line
6. For each commit in chain, extract:
   - commit hash, author, date, message
   - diff snippet showing what changed
   - change_type: 'introduction' (first appearance) | 'modification' | 'refactor'
Return { origin_commit, author, date, message, evolution_chain: [...] }
Handle errors: line doesn't exist, file never in history."

PROMPT 13 (Evolution Chain Parser):
"Create utility function parse_git_log_output(git_log_text):
- Input: raw output from 'git log -L' command
- Parse commit blocks separated by blank lines
- Extract commit hash, author, date from header line (format: '%H|%an|%ae|%ad|%s')
- Extract diff hunks (lines starting with +/-)
- Classify change type based on diff context
Return array of commit objects with metadata and diff."

PROMPT 14 (Trace Endpoint):
"Create POST /api/archaeology/trace endpoint:
- Request: { repo_path, file_path, line_number, issue_type (optional) }
- Calls trace_bug_origin()
- Stores result in GitArchaeology table
- Returns full archaeology data
Add error responses for: line out of range, file not found, git errors."
Validation Checklist:

✅ Can trace line to origin commit
